Angular JS / React JS / Vue JS

라이브러리 / 프레임워크 시작으로~

  • - 라이브러리는 개발자가 필요한것을 빌려와서 쓰는 대상이고 프레임워크는 개발자가 어떻게 개발할 것인지 결정해주는 틀이다.
  • - 프레임워크는 개발자들이 이미 만들어 놓은 함수들의 집합이고, 만들고자 하는 프로그램의 전체적인 틀을 제공해주는 틀이다.

앵귤러(Angular JS)특징

  • 앵귤러는 구글이 만든 단일 페이지 웹 애플리케이션 개발을 위한 자바스크립트 프레임워크이다.
    구글은 브라우저를 만드는 벤더사로 앵귤러 역시 이러한 개발 표준에 따라 개발되었으며 기본적으로 사용하는 모든 것들이 표준이다.
    그래서 HTML 표준 템플릿, 웹 컴포넌트 , 웹 애니메이션 API 등 직접적으로 표준을 따르고 있다.

    Angular JS의 특징은 기존의 프레임워크들과는 달리 DOM조작과 제어가 아닌 데이터 변화에 포커스를 맞춘 방식을 사용하고 있다.
    때문에 자바스크립트에서 작업해야하는 코드는 상당히 간단해 졌고 개발속도도 빨라졌다.
    유지보수 또한 쉬우며 모듈별로 테스트 코드를 작성하는게 쉬어졌다는 장점이 있으며 백엔드와 프론트엔드단을 완전히 분리가 가능하다.
    또한 버전 업그레이드를 통해 서버사이드 랜더링 지원과 타입스크립트를 도입하였다.

    - 서버사이드 렌더링방식
    1. 최초 페이지 접속시 유저가 보는 속도가 빨라짐
    2. 검색엔진 최적화에도 문제 없음
    3. 서버와 클라이언트 사이드 코드를 하나로 합칠 수 있음
    Angular JS 버전 1점때 ~ Angular 2~7 이렇게 불리우고있다.( * Angular는 Node JS 기반으로 작업되어있다. )

리액트(React JS)특징

  • 리액트는 페이스북 자사 서비스의 UI를 효율적으로 만들기 위해 제작한 자바스크립트 라이브러리이다. (프레임워크 라고도함..의견차이)
    가장큰 특징은 MVC (Model View Controller) 패턴에서 컴포넌트를 통해 View를 표현한 것만 집중하고 있다.
    따라서 UI개발자의 경우 리액트를 사용했을 때 큰 메리트를 느낄 수 있다.
    또다른 장점으로는 커스텀마이징으로 나만의 태그를 만들 수 있는 커스텀 태그가 있다.
    이를 통해 웹개발이 가능한 방식을 지원한다.
    기능을 모듈화 할 수 있고 코드를 이해하기 간단하고 쉬울 뿐만 아니라 개발자가 코드를 수정하거나 다시 사용하기 쉽다는 특징을 가지고 있다.

    React 는 다른 JavaScript 프레임웍들이 겪었던, 다량의 데이터셋을 효율적으로 처리하는 것을 목표로 만들어졌다.
    virtual DOM (가상 DOM) 을 사용하고, 변동사항이 있을 때 전체 페이지를 다시 렌더링 하는 방식 대신에, 관련있는 부분만 갱신하는 메카니즘을 사용한다. ( SPA방식 )
  • - XML과 비슷한 JSX를 사용
  • - JSX는 마크업과 로직을 분리하지 않고 XML스타일의 언어를 사용하여 자바스크립트코드에서 직접 마크업을 작성할 수 있다.
    JSX는 정적페이지이기 때문에 컴파일러에서 지속적으로 오류를 잡아줄 수 있다.
    Flow라는 자바스크립트용 유형확인 도구를 사용한다. redux라는 단순한 상태변경 라이브러리가 있다. 이는 프로젝트의 구조를 명확하게 해준다.

어플리케이션 구조

  • AngularJS 와 React 의 또 다른 점은 각 프레임워크를 표현하는 구조의 방식에 있다.
    AngularJS 는 MVC(Model - View - Controller) 구조를 처리하는 데에서부터 시작했고, MVVM(Model - View - ViewModel) 으로도 발전했다.
    반면에 React 는 MVC 의 "V" 에만 집중하며 데이터 표현에만 집중하도록 설계되었고
    다른 어플리케이션 구조적인 부분은 개발자의 선택으로 남겨두었다.
    이런 아키텍처의 유연함 덕분에 생기는 재미있는 점은, AngularJS 어플리케이션의 성능 향상을 위해 일부 컴포넌트(또다른 프레임워크 JS)를 React 로 구성할 수 있다는 것이다.

Angular JS vs React JS

  • 선택은 결국 개발자의 몫이다.
    앵귤러는 MVC를 표방하나 리액트는 그중에서 View에 집중한다는 점이며
    앵귤러는 트렌디한 프레임워크로 재사용 가능한 UI 경험을 제공하고 개발 프로세스에 대한 수준을 향상, 웹과 모바일 그리고 데스크탑, 애플리케이션에 모두 적용이 가능하다는 것이 큰 특징이다.

    AngularJS 와 React 를 선택하는 것....
    React 를 공부하고 사용하는 노력만큼의 잠재적인 성능 향상이 고려되어야 하는가?
    아니면 AngularJS 구조 위에 React 컴포넌트를 올리는 것이 시너지 효과를 가져다 주는가?
    당신의 어플리케이션에 가장 적합한 선택이 필요할 것이다.

Vue JS 특징

  • Vue JS는 Angular JS / React와 다르게 기업소유가 아닌 개인소유자 에반 유(Evan You)이다.
    2013년에 발표되어 GitHub에서도 별을 많이받은 프레임워크이다.
    Vue JS는 Angular의 양방향 데이터 바인딩과 React의 가상 DOM을 가지고 있으며,
    Vue는 React 소스와 달리 배우기쉽고 바로 프로덕션에서 사용할수있습니다.
    대규모 코드베이스 관리에 유용합니다.

React vs Vue JS

  • Vue JS로 작업시에 별도의 변환작업이 없어도 브라우저에서 바로동작이되기때문에, JQuery처럼 쉽게 프로젝트에 적용이 가능합니다.
    React도 가능하나 JSX와 ES6(class, non-mutating array method)에 의존성이 많아 어렵습니다.
    Vue JS에서 새로운 객체를 생성하게되면 getter / setter로 변환하여, Vue 모든 상태를 모니터링하여 변경이 일어날때마다 자동으로 DOM을 다시 랜더링합니다.
    Vue에서는 간단하게 객체변경을 할수있으며, 렌더링 시스템이 React보다 빠르고 더효율적입니다.
    Vue 의 라이브러리 크기 25.6KB / React 라이브러리 (React DOM 37.4KB + React Addons Library 11.4KB) 크기 48.8KB

    큰규모의 프로젝트앱을 만들시에는 React를 사용하야합니다.
    Javascript로 만들어진 React Template은 컴포넌트로 구성하기 쉽고, 소스 재사용성이 높으며 테스트하기 쉽습니다.
    Vue JS 보다 React JS는 보다 정밀한 렌더링 시스템을 가지고있어 렌더링될시 테스팅 도구와 결합하여 테스트하기 수월하며, 유지보수 가능한 코드를 만들수있습니다.
